# 投資調査：バフェット・マンガー・段永平・李録による統合分析

$ARGUMENTS に対して、体系的な投資調査を行う。

## 調査フレームワーク

ウォーレン・バフェット、チャーリー・マンガー、段永平（ドゥアン・ヨンピン）、李録（リー・ルー）の方法論を参照し、次の順序で調査する。

### 事前工程：AI調査の偏りを点検する

調査開始前に、対象企業の**AIによる調査可能性**を評価し、公開情報量に起因する偏りを確認する。

#### 情報量の評価

| 区分 | 特徴 | AI調査で起こりやすい問題 | 対応 |
|---|---|---|---|
| A（情報が豊富） | 上場歴が長く、証券会社や報道のcoverageが厚い | 市場の共通見解を反復しやすく、独自の情報優位が乏しい | 反対仮説を重点確認する。合理的な投資家が買わない理由、見落とされたリスクを探す |
| B（情報が中程度） | 上場後1～3年、coverageが限定的で、一部推定が必要 | 空白をもっともらしい推測で埋め、偽の確実性を作りやすい | 推定値ごとに確度を示し、根拠ある推定と根拠のない補完を分ける |
| C（情報が少ない） | 上場直後、注目度の低い銘柄、新興市場など | 情報不足を「事業が悪い」「理解不能」と誤認しやすい | 第一原理の問いに絞り、限られた一次情報から事業の本質を抽出する |

#### C区分で使う第一原理の問い

公開情報が少ないときは、完全に見える報告を無理に作らず、次を優先する。

1. 顧客は誰か。なぜ支払うのか。代替手段は何か。
2. 再購入・継続利用の要因は、習慣、lock-in、継続的な価値創出のどれか。
3. 十分な資金を持つ競合が、同じ事業を再現できない理由は何か。
4. 経営陣は過去にどの重要判断を行い、その結果から判断力と価値観をどう評価できるか。

#### 偏りの自己点検

- [ ] 確実だと感じる理由は、事業の本質か、資料の量か。
- [ ] 公開資料が半分しかなくても結論は同じか。
- [ ] 分析が市場の共通見解とほぼ同じなら、どこに情報または解釈の優位があるか。
- [ ] 情報は少ないが事業の質は高い可能性を、過小評価していないか。
- [ ] 好意的な物語へ都合の悪い証拠を除外していないか。

情報量区分を報告冒頭へ記載し、最終結論では**AI分析の確度**と**投資対象そのものの予測可能性**を分ける。

### 手順1：データを収集する

> **データ取得規程**：`skills/financial-data.md`に従う。重要な財務データは原則として2つの独立した情報源で確認し、差異が1%を超える場合は明記する。
>
> - 米国株：macrotrendsとstockanalysisを参考にし、SEC提出資料を一次資料とする。
> - 香港株：aastocksとADRデータを参考にし、HKEX開示を一次資料とする。
> - 中国本土A株：Eastmoney（东方财富）を参考にし、CNINFO（巨潮资讯）と法定開示を確認する。

TaskツールでバックグラウンドAgentを起動し、WebSearchと一次資料から次を収集する。

1. 売上構成：直近会計年度と直近4四半期のsegment別売上高、成長率、売上総利益率。
2. 財務指標：直近5年の売上高、純利益、売上総利益率、営業利益率、FCF、現金・短期投資。
3. 競争環境：市場シェア、主要競合、シェア推移。
4. 事業モデルと経済的な堀：競争優位の源泉。
5. 技術力：中核技術、研究開発投資、技術依存関係。
6. 経営陣：創業者・CEOの経歴、持株、重要な意思決定。
7. 業界見通し：TAM、成長予測、成熟度。
8. リスク：地政学、規制、供給網、顧客集中、技術変化。
9. 現在のバリュエーション：時価総額、PER、PSR、PEG、EV/Revenueなど。
10. 強気・弱気双方の主要論点。

各情報に公表日、対象期間、通貨、単位、会計基準を付ける。企業側の説明、市場予想、独自推定を分ける。

#### データのクロスチェック

収集後、`tools/financial_rigor.py`で重要数値を機械的に検証し、LLMの暗算だけで確定しない。

**必ず確認する項目**：

- 発行済株式数：取引所、企業開示、信頼できる金融データサービスなど、少なくとも2情報源。
- 現在株価と時価総額：`株価 × 発行済株式数`を計算し、公表時価総額と比較する。
- 直近会計年度の売上高と純利益：年次報告書と少なくとも1つの独立した情報源。
- 現金とnet cash：現金、短期投資、総負債、リース負債などの定義を確認する。
- 経営陣の持株：経済的持分と議決権を分け、dual-class構造を確認する。

**必須コマンド**：

手順1：時価総額の検算。精密な十進計算を使う。

```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-market-cap \
  --price {株価} --shares {発行済株式数} --reported {報告時価総額} --currency {通貨}
```

手順2：重要データの複数情報源比較。

```bash
python3 ~/ai-berkshire/tools/financial_rigor.py cross-validate \
  --field {項目名} --values '{"情報源1": 数値, "情報源2": 数値}' --unit {単位}
```

売上高、純利益、現金・短期投資などへ個別に実行する。

手順3：PER、PBR、ROE、FCF Yieldなどの検算。

```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation \
  --price {株価} --eps {EPS} --bvps {1株当たり純資産} --fcf-per-share {1株当たりFCF} --dividend {1株当たり配当}
```

**検証規則**：

1. 重要データは原則として2つ以上の独立した情報源で確認する。
2. 差異がある場合は、法定開示、取引所、企業公式資料を優先し、差異の原因を説明する。
3. 計算を伴う重要指標はツールで検算する。
4. ツール出力を報告付録「重要データのクロスチェック記録」へ保存する。
5. ツールが大きな差異を報告した場合は、単位、通貨、期間、株式数、会計定義を解決するまで分析を確定しない。

**誤りやすい点**：

- 時価総額の単位：億HKD、億元（CNY）、億USDを混同しない。
- FCFの定義：設備投資にリース、買収、capitalized developmentを含むか確認する。
- 負債の定義：営業リース負債を含むか確認する。
- 持株比率：dual-class企業では経済的持分と議決権が異なる。
- ADR：ADR比率と原株換算を確認する。

### 手順2：事業の本質 — 段永平の「正しい事業」

- 事業の本質を一文で定義する。
- 売上構成を表または図で分解する。
- 5年間の収益性推移を示す。
- 事業モデルを整理する。一回販売か継続課金・再購入か、hardwareかsoftwareかplatformか。
- ecosystemへの定着、顧客の切替コストを評価する。
- 売上総利益率を競合と比較し、高低の理由を説明する。
- operating leverageを確認する。

**段永平式の問い**：この事業は何が優れているのか。一文だけで説明すると何か。

### 手順3：経済的な堀 — バフェットの競争優位性

| 堀の種類 | 検証方法 |
|---|---|
| ブランド・価格決定力 | 販売数量や継続率を大きく損なわず値上げできるか |
| 切替コスト | 顧客が競合へ移る際の金銭・時間・運用上の負担は何か |
| ネットワーク効果 | 利用者増加が他の利用者の価値を高めるか |
| 規模の経済 | 規模拡大が単位原価、調達、distributionへどの程度効くか |
| 技術・特許障壁 | 優位が何年続き、競合が再現するために何を必要とするか |

過去5年で堀が拡大・安定・縮小のどれかを判断し、今後5年の方向をscenarioで示す。

**バフェット式の問い**：10年後もこの堀は残るか。何がそれを破壊し得るか。

### 手順4：逆向き思考とリスク — マンガーの「反対から考える」

- 企業が失敗する経路をできるだけ列挙し、発生可能性、影響、先行指標を表にする。
- 類似した立場にあった過去の企業と、その後の結果を調べる。
- ネットワーク効果、技術採用曲線、競争ゲーム、incentiveなど複数のモデルで検証する。
- 物語への偏り、anchoring、生存者バイアス、確証バイアスを点検する。
- 弱気論と空売り側の主張を収集し、事実と推測を分ける。

**マンガー式の問い**：最も誤りやすい前提は何か。合理的な投資家が買わない、または売り持ちにする理由は何か。

### 手順5：経営陣 — 段永平の「正しい人」とバフェットの誠実性

- CEO・創業者の重要判断を、時期、判断、結果、評価で整理する。
- 資本配分を確認する。研究開発、M&A、自社株買い、配当、債務削減の成果を比較する。
- 経営陣の持株、報酬、売却履歴から株主との利害一致を確認する。
- 組織の安定性、key person risk、successionを評価する。
- 企業文化が実際の制度と行動へ反映されているかを確認する。

**段永平式の問い**：CEOが退任しても、競争力と文化を維持できるか。

### 手順6：業界と長期変化 — 李録の文明進化フレームワーク

- 業界が社会や技術の大きなparadigm shiftに位置するかを判断する。
- 蒸気機関、電力、インターネット、AIなど過去の技術革新との類似点と相違点を示す。
- TAMの成長曲線、penetration、上限を検討する。
- 企業がvalue chainのどこに位置し、価値をどこで獲得するかを示す。
- 技術経路の不確実性と代替技術を評価する。
- 顧客・supplier集中を確認する。

**李録式の問い**：20年後から振り返ったとき、この企業は時代の中核的な勝者か、一時的な先行者か。

### 手順7：バリュエーションと安全余裕 — バフェットの内在価値と段永平の価格規律

- 現在の市場評価を、検算済みの指標で示す。
- reverse DCFで、現在価格が暗黙に要求する成長率と利益率を推定する。
- 楽観・基準・悲観の3scenarioをツールで計算する。

```bash
python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario \
  --price {株価} --eps {EPS} --shares {発行済株式数} \
  --growth {楽観成長率} {基準成長率} {悲観成長率} \
  --pe {楽観PER} {基準PER} {悲観PER} --years 3 --currency {通貨}
```

- 自社の過去の評価レンジと比較する。
- 競合の評価指標と比較する。
- 成長率、margin、株式数、評価倍率の仮定を明記する。
- 一点の目標株価ではなく、scenario別の価値範囲と安全余裕を示す。

**段永平式の問い**：市場が明日から5年間閉鎖されても、この価格で保有したいか。

## 出力要件

1. すべての重要判断にデータと出典を付ける。
2. 主要数値はMarkdown表で示す。
3. 各moduleの末尾に対応する「問い」を置く。
4. 完全な報告を`reports/{会社名}/{会社名}-research-{YYYYMMDD}.md`へ必ずファイル保存する。会話応答への出力だけで終わらせず、書き込みツールまたはシェルコマンドでディレクトリ（`reports/{会社名}`）を作成し実際にファイルを保存すること。
5. 結論は、調査継続、候補、見送り、判断保留などを明確に示し、根拠と反対意見を併記する。
6. バリュエーションは具体的な範囲を示すが、仮定と不確実性を明記する。
7. 報告冒頭に情報量区分A/B/Cと、AI調査の限界を記載する。
8. 報告末尾でAI分析の確度と投資対象の予測可能性を分け、十分なデータに基づく結論と、限られた情報からの推論を明示する。
9. C区分では「一次確認が必要な問い」を列挙し、製品利用、顧客確認、供給網調査などAIが代替できない確認方法を示す。
10. 調査日、データ基準日、対象市場、使用通貨、会計基準を記載する。
11. 本資料は調査・学習用であり、投資助言ではないと明記する。
12. 最終出力は共通locale設定に従い、自然な日本語の「だ・である調」とする。

## データ抽出監査

報告を書いた後、公開可能と判断する前にデータ抽出監査を行う。

手順1：15%の無作為抽出候補を生成する。

```bash
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report <報告ファイルパス>
```

出力JSONの各項目には`fetched_value`を入力する欄がある。

手順2：抽出された各データ点を`skills/financial-data.md`に従って再取得する。`fetched_value`、`fetched_source`、`fetched_value2`、`fetched_source2`を埋める。

手順3：判定を出す。

```bash
python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<入力済みJSON>' \
  --report <報告ファイル名>
```

- **合格**：すべての抽出点の差異が1%以下である。
- **差戻し**：いずれかの差異が1%を超える。該当値の通貨、単位、期間、会計定義、出典を修正し、再度監査する。

監査ツールの合格だけで報告全体の正確性が保証されたとは判断しない。抽出漏れ、定性的主張、株式分割、会計期間、TTMと年度実績の混同もセルフレビューする。
