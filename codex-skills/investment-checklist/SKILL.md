# バフェット流・購入前投資チェックリスト

$ARGUMENTS に対して、バフェット流の価値投資チェックリストを実行する。

**対応する入力形式**：1社または複数社を、カンマ、読点、空白で区切る。例：`Tencent, Kweichow Moutai, NVIDIA`または`NVDA AAPL MSFT`

## 実行手順

### 手順1：入力を解析し、対象企業を特定する

$ARGUMENTS から企業名またはtickerをすべて抽出する。各社について次を確認する。

- 正式な会社名、ticker、上場取引所。
- 未上場の場合は「未上場」と明記し、利用可能な間接的な投資手段が公に確認できる場合だけ簡潔に説明する。上場株向けの完全なチェックリストは実行しない。
- 同名企業や複数上場がある場合は、対象証券を特定する。

### 手順1.5：AI調査の偏りを警告する

各社の**情報量**をA/B/Cで簡易評価し、報告に明記する。

| 区分 | 判定基準 | チェックリストへの反映 |
|---|---|---|
| A | 上場歴が長く、法定開示と時系列データが十分にある | 通常どおり実行する。ただし、情報が多いことを事業の確実性と取り違えない |
| B | データが限られ、一部の推定が必要である | 推定指標ごとに確度を示し、「優れた事業」の判断へデータ品質を反映する |
| C | 公開情報が著しく少ない | 6項目を無理に埋めず、「情報不足で判定不能」とする。検証可能な問いを優先する |

**中核原則**：このチェックリストは、悪い選択肢を早期に除外するためのものである。C区分の「情報不足」は通過でも不通過でもない。「判断保留。一次情報が必要」と明記し、AIが表を埋められないことだけを理由に否決しない。

段永平（ドゥアン・ヨンピン）の考え方では、「理解できない」には、事業が複雑で本当に理解困難な場合と、まだ十分に調べていない場合がある。公開情報の少なさと事業の理解不能を混同しない。

### 手順2：データを並行収集する

Taskツールで**企業ごとに**独立したバックグラウンドAgentを起動し、すべてを並行実行する。各Agentは一次資料を優先して次を収集する。

1. **収益性**：ROEの5～10年推移、売上総利益率、純利益率、フリーキャッシュフロー。
2. **バリュエーション**：現在株価、時価総額、PER（TTM）、予想PER、PBR、配当利回り。
3. **成長**：直近3年の売上高と利益の成長率。
4. **財務健全性**：負債、設備投資負担、現金、net cashまたはnet debt。
5. **競争環境**：市場シェア、主要競合、シェア推移。
6. **経済的な堀**：ブランド、切替コスト、ネットワーク効果、規模の経済、技術障壁を裏づける具体的証拠。
7. **経営陣**：CEOの経歴、重要な意思決定、保有株式、資本配分の実績。
8. **直近の変化**：過去6か月の決算、M&A、規制、経営陣交代など。

重要な財務データは`skills/financial-data.md`に従い、原則として2つの独立した情報源で確認する。通貨、単位、会計期間、会計基準、データ基準日を付ける。

### 手順3：企業ごとに6つの関門を確認する

---

#### 関門1：事業を理解できるか（能力の輪）

必ず次に答える。

- [ ] この会社が何によって利益を得るかを一文で説明できるか。
- [ ] 10年後も、どの事業を営んでいる可能性が高いか。
- [ ] 成否を決める重要変数は何か。
- [ ] 業界理解は一次資料と深い調査に基づくか、それとも伝聞にすぎないか。

**評価基準**（★1～5）：

- ★★★★★：事業モデルが極めて明確で、10年後の主要事業を高い確度で想定できる。
- ★★★★☆：事業モデルは明確だが、理解に一定の専門知識を要する。
- ★★★☆☆：収益構造は理解できるが、技術や業界の変化が速く、10年後の確度は中程度である。
- ★★☆☆☆：事業が複雑、または業界構造が急変中で、将来像を描きにくい。
- ★☆☆☆☆：能力の輪の外にある。

**即時否決条件**：利益が生まれる仕組みを説明できない場合は、「能力の輪の外」として以降の判断を保留する。もっとも、情報不足で説明できない場合と、構造的に理解できない場合は区別する。

---

#### 関門2：優れた事業か（経済性）

定性的な印象だけでなく、重要指標をツールで計算する。

```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation \
  --price {株価} --eps {EPS} --bvps {1株当たり純資産} --fcf-per-share {1株当たりFCF} --dividend {1株当たり配当}
```

| 指標 | 対象企業の数値 | 参考基準 | 判定 |
|---|---:|---|---|
| ROE（5年平均） | | 15%超は良好、20%超は非常に高い | |
| 売上総利益率 | | 40%超は価格決定力を示唆するが、業種差を考慮する | |
| フリーキャッシュフロー | | 継続的にプラスで、純利益との乖離が説明可能 | |
| 設備投資負担 | | 同じ競争力なら資本負担が小さい方が望ましい | |
| 負債 | | 有利子負債 / 純利益が3年未満を一つの参考とする | |

業種、会計基準、景気循環を考慮し、閾値を機械的に適用しない。

**評価基準**（★1～5）：

- ★★★★★：高ROE、高い粗利益、強いFCF、低い追加資本需要、低負債をすべて満たす。
- ★★★★☆：5項目中4項目を満たす。
- ★★★☆☆：3項目を満たす。
- ★★☆☆☆：2項目以下、または重要指標が悪化している。
- ★☆☆☆☆：多くの項目が基準未達、またはFCFが構造的にマイナスである。

---

#### 関門3：経済的な堀は十分か（競争優位性）

| 堀の種類 | 有無 | 具体的な証拠 | 拡大・安定・縮小 |
|---|---|---|---|
| ブランド・価格決定力 | | | |
| 切替コスト | | | |
| ネットワーク効果 | | | |
| コスト・規模優位 | | | |
| 技術・特許障壁 | | | |

追加の問い：十分な資金を持つ有力競合が参入しても、同等の事業を容易に再現できない理由は何か。固定額の「100億」は通貨と業界規模を確認し、必要に応じて現実的な参入資金へ置き換える。

**評価基準**（★1～5）：

- ★★★★★：複数の堀が重なり、さらに強くなっている。
- ★★★★☆：少なくとも一つの強い堀があり、安定している。
- ★★★☆☆：堀はあるが浅い、または方向が不明である。
- ★★☆☆☆：堀が明確に侵食されている。
- ★☆☆☆☆：持続的な堀を確認できない。

---

#### 関門4：経営陣を信頼できるか（人的要因）

| 確認項目 | 評価 |
|---|---|
| 誠実性：過去の約束と実績 | |
| 資本配分：自社株買い、配当、M&A、投資 | |
| 株主との利害一致：持株、報酬設計 | |
| 所有者意識：創業者か職業経営者かだけでなく、実際の行動 | |
| ガバナンス：関連当事者取引、のれん、監査、取締役会 | |
| CEO退任後も組織が機能するか | |

**評価基準**（★1～5）：

- ★★★★★：長期的な所有者意識、優れた資本配分、株主との強い利害一致がある。
- ★★★★☆：経営陣は優れているが、軽微な懸念がある。
- ★★★☆☆：概ね適格だが、ガバナンスまたは継承に懸念がある。
- ★★☆☆☆：誠実性、資本配分、統治のいずれかに重大な疑義がある。
- ★☆☆☆☆：重大な不誠実行為を確認した。即時否決条件とする。

#### 関門5：価格に十分な安全余裕があるか

| 指標 | 数値 | 過去分布 | 判定 |
|---|---:|---:|---|
| PER（TTM） | | | |
| 予想PER | | | |
| PBR | | | |
| 配当利回り | | | |
| FCF Yield | | | |

**重要な計算は暗算せず、ツールで行う。**

```bash
python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario \
  --price {株価} --eps {EPS} --shares {発行済株式数} \
  --growth {楽観} {基準} {悲観} --pe {楽観PER} {基準PER} {悲観PER} --currency {通貨}
```

- 楽観・基準・悲観シナリオの価値範囲を、ツール出力から示す。
- 判断が誤っていた場合の下方余地を試算する。
- 株価が半減したとき、追加投資できる根拠と資金余力があるかを確認する。

**評価基準**（★1～5）：

- ★★★★★：推定内在価値の50%以下で、極めて大きな安全余裕がある。
- ★★★★☆：推定内在価値の約70%以下で、良好な安全余裕がある。
- ★★★☆☆：概ね適正価値で、安全余裕は限定的である。
- ★★☆☆☆：割高で、安全余裕が不足する。
- ★☆☆☆☆：悲観シナリオを含めても著しく割高である。

内在価値は一点ではなく範囲として扱い、成長率、terminal multiple、株式数、通貨などの仮定を記載する。

---

#### 関門6：positionと意思決定の規律

次の感情的な兆候を確認する。

- FOMOだけで買おうとしていないか。
- 他者の推薦だけが購入理由になっていないか。
- 5年間取引停止になっても保有できるか。
- 購入仮説を200字以内で具体的に書けるか。
- 投資額が失われても生活や重要な資金計画を損なわないか。

---

### 手順4：ミラーテスト

各社について次を埋める。

> 「私は ___（通貨）で ___社を購入候補とする。理由は次のとおりである。
> 1. 事業の本質は___であり、収益構造を理解している／理解できていない。
> 2. 経済的な堀は___であり、拡大／安定／縮小している。
> 3. 経営陣は___であり、信頼できる／できない。
> 4. 現在価格は推定内在価値の約___%であり、安全余裕がある／ない。
> 5. 仮説が外れた場合の下方リスクは___であり、許容できる／できない。」

**5文を具体的に書けない場合は購入判断を通過させない。** 「通過」「不通過」「判定保留」を明記する。

---

### 手順5：即時否決条件を確認する

各社について確認し、該当する場合は「否決」とする。ただし、事実確認が不十分なら「疑義」「要確認」とし、断定しない。

- [ ] 利益が生まれる仕組みを説明できない。
- [ ] 3年連続でFCFがマイナスで、改善経路を確認できない。
- [ ] 経営陣に重大な誠実性の問題がある。
- [ ] 競争優位が不可逆的に侵食されている。
- [ ] 利益が、次の買い手がより高く買うことだけに依存する。
- [ ] 投資額が全損した場合の影響を許容できない。
- [ ] 主な購入理由が「皆が買っている」「最近上昇している」である。
- [ ] 200字以内で購入理由を説明できない。

|---|---|---|---|---|---|---|---|
| | | ★☆☆☆☆ | ★☆☆☆☆ | ★☆☆☆☆ | ★☆☆☆☆ | ★☆☆☆☆ | |

### 手順7：最終結論を出し、ファイルへ保存する

各社について曖昧さを隠さず、次のいずれかを示す。

- ✅ **チェックリスト通過**（6関門中X）：深掘り調査へ進める。
- ❌ **不通過**：抵触した即時否決条件を示す。
- ❓ **判断保留**：重要な争点、情報不足、追加確認事項を示す。
- N/A：未上場、対象証券を特定できない、または直接投資できない。

完全な報告を、会社単位では`reports/{会社名}/{会社名}-checklist-{YYYYMMDD}.md`、複数社比較では`reports/checklist-comparison-{YYYYMMDD}.md`へ保存する。

## 出力要件

1. 各社を独立した章にし、6関門の評価表、中核データ、3～5件の主要リスク、ミラーテスト、明確な結論を含める。
2. 複数社の場合は最後に比較表を付ける。
3. 評価は★1～5で示し、半星を使わない。
4. 各データに出典、対象期間、基準日を付け、推定値には「推定」と記載する。
5. 結論では「投資の第一原則は元本を守ることである」というバフェットの趣旨に沿って、安全余裕と下方リスクを再確認する。
6. 文体は共通locale設定に従う。簡潔で率直な「だ・である調」とし、バフェット、マンガー、段永平（ドゥアン・ヨンピン）の公開発言を使う場合は、原意を損なわず、確認できない引用を捏造しない。
7. 本資料は調査・学習用であり、投資助言ではないと明記する。

## 重要原則

- **見逃しても、理解しないまま買わない**：目的は悪い選択肢を除外することであり、すべての機会を捕捉することではない。
- **能力の輪に誠実である**：分からないことは「分からない」とする。
- **安全余裕を軽視しない**：優れた会社でも高過ぎる価格で買えば損失になり得る。
- **ミラーテストを省略しない**：購入理由を具体的に説明できなければ、購入判断を通過させない。
- **事実、推定、仮定、評価を区別する**：会社説明や市場予想を確認済み事実と混同しない。
