# 業界投資調査：value chain全体と主要企業を分析する

$ARGUMENTS の業界または投資テーマについて、体系的なvalue chain調査を行う。

## 調査目的

一つの投資テーマまたは因果仮説から出発し、次を行う。

1. 投資仮説を構成する因果関係を一つずつ検証する。
2. value chain全体を可視化する。
3. 中国本土A株、香港株、米国株、日本株、韓国株、欧州株、豪州株など、主要市場の上場企業を広く探索する。
4. 各segmentの主要企業を、バフェット、マンガー、段永平（ドゥアン・ヨンピン）、李録（リー・ルー）の観点で分析する。
5. 業界全体のリスクと、複数の資産配分案を示す。

---

## 手順1：投資仮説の因果chainを作り、検証する

### 1.1 因果chainを描く

基礎的な変化から受益企業までを矢印で示す。

```text
基礎的な変化 A
    → 需要 B を生む
        → 供給制約または不可欠な機能 C が発生する
            → value chain D の企業が利益を獲得する
```

各矢印は別々の仮説である。最後の結論が魅力的でも、中間の矢印が成立しなければ投資仮説は弱い。

### 1.2 各矢印を反証可能な形へする

| 接続 | 中核仮説 | 検証方法 | 情報源 | 反証条件 |
|---|---|---|---|---|
| A→B | | 公的統計、顧客支出、企業guidance | | |
| B→C | | 供給能力、lead time、価格、代替技術 | | |
| C→D | | 契約、margin、利益配分、参入状況 | | |

予測値は予測主体、基準年、forecast期間、scenarioを明記する。

### 1.3 既に起きた検証事象を探す

単なる市場予測ではなく、次のような実行済み・公表済み事象を列挙する。

- 契約締結、発注、稼働開始、価格改定。
- 政策の成立・施行、license付与。
- 供給不足、lead time、稼働率、在庫の実測値。
- 顧客の設備投資、導入率、解約率。

公表日と事象発生日を区別し、会社側の説明と外部確認を分ける。

## 手順2：value chain全体を描く

### 2.1 構造を分解する

業界を上流、中流、下流、補助機能へ分ける。

```text
上流：原材料・資源 → 精製・素材加工
中流：中核装置 → component → system integration → engineering・construction
下流：運営・service → distribution → 最終顧客
補助：testing・certification → maintenance → software・data → financing
```

実際の業界構造に合わせ、無理にこの4区分へ押し込めない。

### 2.2 各segmentの事業特性を示す

| segment | 事業モデル | 売上総利益率の範囲 | 競争構造 | 主な障壁 | cycle |
|---|---|---:|---|---|---|
| | resource / equipment / service / recurring fee | | monopoly / oligopoly / fragmented | resource / license / technology / scale / switching cost | 強 / 中 / 弱 |

利益率は同じ会計基準と期間で比較する。業界内の利益poolがどこにあり、なぜ維持されるかを説明する。

### 2.3 bottleneckを特定する

供給が最も制約され、代替が難しく、価格決定力を持つsegmentを探す。ただし、現在の高marginが一時的な不足だけに依存する場合は、能力増強後のnormalizationを検討する。

各bottleneckについて次を示す。

- 制約となる資源、技術、認証、人材、時間。
- 供給拡大に必要な期間と資本。
- 顧客が代替・内製化できる可能性。
- 価格上昇が需要破壊または政策介入を招く水準。

## AI調査の偏りを点検する

業界調査では、公開情報量の偏りが企業選定へ直接影響する。

| 偏り | 起こり方 | 対応 |
|---|---|---|
| 成熟業界への偏り | 銀行、energy、consumerなどは資料が多く、確実に見える | 確実性を資料量ではなく、事業構造とcash flowから判断する |
| 新興業界の過小評価 | AI application、synthetic biologyなどは履歴が短く、保守的になりやすい | 終局scenarioと反証条件を使い、現在値だけで判断しない |
| 大型株への偏り | 大企業は記事と分析が多く、候補に残りやすい | 小型株も同じ中核問いで評価し、情報不足は別軸で示す |
| 上場企業への偏り | 非上場の中核supplierや将来の競合を見落とす | 重要な非上場企業も調べ、「直接投資不可」「将来IPO候補」などと区別する |
| 英語情報への偏り | 英語資料が多い企業を過大評価し、中国・アジア企業を見落とす | 日本語、英語、中国語など複数言語で検索し、一次資料を優先する |

反偏見の実行規則：

1. 各segmentで、著名企業だけでなく、専門性の高い小型企業や地域企業を能動的に探す。
2. 情報が少ない企業を、報告の短さだけで低評価にしない。事業の本質、堀、経営陣を可能な範囲で評価する。
3. 各社に情報量A/B/Cを付け、投資評価とデータ確度を分ける。
4. 中国語の正式名称と検索語を使ってよいが、最終説明は自然な日本語とする。

## 手順3：世界の関連企業を探索する

TaskツールでバックグラウンドAgentを起動し、次を調べる。

- 米国：NYSE、NASDAQ、NYSE American。
- 中国本土：上海、深圳、北京などのA株。
- 香港：HKEX上場企業。
- 日本、韓国、欧州、豪州、Canadaなどの主要市場。
- 業界ETF、closed-end fund、trust。
- 重要な非上場企業。

### 各社で収集する項目

- 正式名称。必要に応じて原語・英語名を併記する。
- tickerと取引所。
- 時価総額、通貨、基準日時。
- value chain上の役割を示す一文。
- pure playか、多角化企業の一部か。関連売上比率が分かれば示す。
- 所属segment。
- 上場証券の種類、ADR比率、流動性など投資手段上の注意。
- 情報量A/B/C。

### 分類

segmentごとに表を作り、次の区分を参考にする。

- **Tier 1**：規模が大きく、関連売上比率が高い主要企業。
- **Tier 2**：中規模で、pure playまたはsegment leader。
- **Tier 3**：小規模、開発段階、収益前など、高い不確実性とoptionalityを持つ企業。
- **Tier 4**：多角化企業の一部として関連事業を持つ企業。

Tierは「投資推奨度」ではなく、規模、pure play度、事業段階の分類であると明記する。

## 手順4：各segmentの主要企業を分析する

Tier 1とTier 2を中心に、各segmentから少なくとも2～3社を比較する。Tier 3とTier 4は重要性に応じて簡潔に扱う。

### 4.1 事業の本質 — 段永平

- value chain上の役割を一文で定義する。
- 売上構成、成長率、関連事業比率。
- 売上総利益率、純利益率と推移。
- cash flowと設備投資負担。

**問い**：これは優れた事業か。利益が生まれる仕組みと持続条件は何か。

### 4.2 経済的な堀 — バフェット

| 堀 | 強さ（★1～5） | 証拠 | 方向 |
|---|---|---|---|
| ブランド・価格決定力 | | | |
| 切替コスト | | | |
| ネットワーク効果 | | | |
| 規模・cost優位 | | | |
| 技術・license・認証 | | | |

**問い**：10年後もこの堀は残るか。顧客の内製化や代替技術で弱まらないか。

### 4.3 リスク — マンガー

- 最も現実的な失敗経路は何か。
- 悲観scenarioで利益と企業価値はどこまで低下し得るか。
- 合理的な投資家が買わない理由は何か。
- 高い現在marginが競争を呼び込まないか。

### 4.4 経営陣 — 段永平とバフェット

- CEO、創業者、支配株主。
- 過去の重要判断と資本配分。
- 経済的持分、議決権、報酬。
- 誠実性、governance、succession。
- A/B/Cは情報量区分と混同しないよう、経営陣評価の意味を明記する。

### 4.5 バリュエーションのsnapshot

- PER、PSR、EV/EBITDA、FCF Yieldなど、業種に適した指標。
- 自社の過去レンジと同segment企業との比較。
- 現在評価を「割高」「おおむね妥当」「割安」「判定不能」のいずれかで示し、基準日と仮定を付ける。
- 会計基準、通貨、利益cycle、net cashの差を調整する。

### 4.6 調査上の優先度

★1～5を使ってもよいが、売買指示ではなく調査優先度として扱う。

- ★★★★★：中核候補。事業の質、堀、評価を深掘りする価値が高い。
- ★★★★☆：有力候補。重要な不確実性を確認する。
- ★★★☆☆：監視候補。
- ★★☆☆☆：高いoptionalityと高い損失リスクを持つ。
- ★☆☆☆☆：現時点では優先度が低い、または即時否決条件がある。

## 手順5：業界全体のリスクを評価する

### 5.1 systemic risk

| リスク | 発生可能性 | 影響 | 先行指標 | 対応 |
|---|---|---|---|---|
| 因果chainの中間仮説が崩れる | | | | |
| 代替技術が普及する | | | | |
| 政策・規制が変わる | | | | |
| 需要cycleが反転する | | | | |
| バリュエーションが正常化する | | | | |
| 能力増強で供給不足が解消する | | | | |

### 5.2 歴史的類似例

類似する過去の投資テーマを選び、表面的な一致だけでなく相違点も示す。

- 類似業界は何か。
- 最終的に価値を獲得したのは上流、中流、下流のどこか。
- 多数の参入企業と投資家はどうなったか。
- 現在の業界に適用できる点と、適用できない点は何か。

### 5.3 認知bias

- narrative bias：因果chainがきれい過ぎないか。
- anchoring：直近の株価上昇、価格高騰、高い成長率へ固定されていないか。
- herding：皆が注目していることを需要の持続性と混同していないか。
- base-rate neglect：類似する新規産業で、最終的に何社が利益を残したか。

- 最も近い過去の技術革新は何か。重要な相違は何か。
- 10～20年後の需要、競争、利益poolの終局scenarioは何か。

**問い**：現在の高成長が終わった後も、誰が持続的にcash flowを獲得するのか。

## データ監査

報告作成後、公開可能と判断する前に抽出監査を行う。

```bash
# 手順1：15%の抽出候補を作る
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report <報告ファイルパス>

# 手順2：skills/financial-data.mdに従い、抽出項目を再取得する

# 手順3：判定する
python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<入力済みJSON>' \
  --report <報告ファイル名>
```

- **合格**：抽出された数値が許容差内である。
- **差戻し**：不一致を修正し、再監査する。

監査合格だけで網羅性や定性的判断の正確性が保証されたとは扱わず、企業一覧の漏れ、通貨換算、会計期間、ETF構成、非上場企業の状態も確認する。
