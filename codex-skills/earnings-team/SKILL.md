# 決算精読チーム：4つの投資観点と編集・読者確認を統合する

$ARGUMENTS に対して、複数Agentによる決算精読を行う。4つの投資観点で一次資料を並行分析し、Team Leadが統合した後、Editor AgentとReader Agentで公開記事として仕上げる。

**対応する入力形式**：`会社名 期間`。例：`Tencent 2025Q4`、`PDD 2025年次報告書`、`Meituan 最新`

## 設計思想

良い決算分析には二つの条件がある。

1. **将来を考えるために、自分が決算を理解できること**：異なる4観点で同じ一次資料を読み、相互に反証する。
2. **読者が事業価値の変化を理解できること**：編集と読者確認で、専門性を落とさず可読性を高める。

工程は3段階である。

- **Phase 1・調査**：段永平の事業観、バフェットの財務品質、マンガーの競争・逆向き思考、李録の永久損失riskを参考に、4Agentが並行精読する。
- **Phase 2・統合**：Team Leadが共通点、矛盾、見落としを抽出し、研究底稿を作る。
- **Phase 3・公開準備**：Editor Agentが記事へ再構成し、Reader Agentが理解可能性と信頼性を確認し、Team Leadが定稿する。

## Phase 1：4Agentによる並行調査

### 手順1：一次資料を取得する

Agentツールで複数のAgentを並行起動し、次を取得する。

| 資料 | 主な取得元 | 優先度 |
|---|---|---|
| 法定開示・決算原文 | 企業IR、SEC EDGAR、HKEXnews（披露易）、CNINFO（巨潮资讯） | 最優先 |
| earnings call transcript・録音 | 企業IR、必要に応じてSeeking Alpha、雪球（Xueqiu） | 最優先 |
| 株主向け書簡 | 年次報告書 | 年次決算では高い |
| 前回決算・前回call | 同上 | 過去の約束追跡に必須 |
| 市場予想 | 出所と集計時点を確認できるservice | 補助 |

#### 資料入手区分

| 区分 | 状況 | 分析への影響 |
|---|---|---|
| A | 完全な原文を取得した | 全工程を実行する |
| B | 原文の一部と信頼できる二次情報を取得した | 二次情報部分を明記し、注記分析の確度を下げる |
| C | 報道・data siteの要約だけである | 中核数値へ絞り、注記や語調を推測しない。「一次資料不足」とする |

資料区分を全Agentへ伝える。各資料の公表日、対象期間、URLまたは取得元を記録する。

### 手順2：チーム構成を示す

| 段階 | 役割 | 参照する観点 | 中核Task |
|---|---|---|---|
| 調査 | **Team Lead** | 統合 | 調整、矛盾整理、定稿 |
| 調査 | business-interpreter | 段永平 | 事業の質は改善したか |
| 調査 | financial-auditor | バフェット | 利益はcashへ変わっているか |
| 調査 | competition-analyst | マンガー | 競争構造はどう変化したか |
| 調査 | risk-scout | 李録 | 永久損失へつながるsignalはあるか |
| 公開 | editor | 長文編集 | 研究底稿を読みやすい記事へ再構成する |
| 公開 | reader-reviewer | 想定読者 | 理解、信頼、行動可能性を確認する |

### 手順3：4Agentを同時に起動する

AgentまたはTaskツールを**同一メッセージ内で4件並行起動**する。各Agentは独立して一次資料を読み、完了後にTeam Leadへ完全な報告を送る。

### Agent 1：事業の本質 — 段永平の観点

**中核問い：この決算が示す事業の質は、改善、横ばい、悪化のどれか。**

#### 1. 売上構成

- segment別・地域別売上高と成長率。
- 加速・減速した事業。
- 成長が数量、価格、mix、買収、為替のどれに起因するか。
- 各segmentの変化が、どの事業logicを示すか。

#### 2. 顧客価値

- DAU、MAU、paying user、subscriberなど、事業に適したKPI。
- 利用時間、ARPU、retention、churn、同店売上など品質指標。
- 顧客が得る価値と、代替手段に対する優位が強まったか。

#### 3. 経済的な堀

- 売上総利益率と価格決定力。
- market shareと競争優位。
- 切替コスト、network effect、scale advantageの変化。
- 増収が値引きや販売費増加に依存していないか。

#### 4. 優れた事業の条件

- 差別化、価格決定力、持続的競争優位。
- 資本集約度が上がったか下がったか。
- 会社がなくなった場合、顧客が強く困る理由はあるか。

#### 5. productと顧客への理解

- 経営陣がproduct、顧客、trade-offを具体的に説明しているか。
- 抽象的なcorporate languageへ逃げていないか。
- 今回の決算にproduct insightまたは顧客との乖離が現れているか。

**出力**：各項目を🟢改善 / 🟡横ばい / 🔴悪化 / ⚪判定不能で示し、最後に段永平の事業観からの要約を記載する。実在しない本人発言を引用しない。

---

### Agent 2：財務品質 — バフェットの観点

**中核問い：報告利益はcashへ変わっているか。財務耐久性と安全余裕はどう変わったか。**

#### 1. 中核数値

- 売上高、売上総利益、営業利益、純利益、EPS。
- GAAPまたはIFRSとNon-GAAPの両方。
- 調整項目、差額、差の拡大・縮小。
- 重要数値を2つ以上の独立情報源で確認する。

```bash
python3 ~/ai-berkshire/tools/financial_rigor.py cross-validate \
  --field "revenue" \
  --values '{"情報源1": 数値1, "情報源2": 数値2}' \
  --unit {通貨・単位}
```

実行前に実装または`--help`を確認し、CLI仕様へ合わせる。

#### 2. cash flow

- 営業キャッシュフロー / 純利益。100%超を良好、80%未満を警戒の参考とし、運転資本と業種差を説明する。
- FCF = 営業キャッシュフロー - capital expenditure。
- 維持投資と成長投資。開示がなければ推定とする。
- 自社株買い、配当、買収、債務返済。

#### 3. 利益品質

- 売上債権の増加率と売上高成長率。
- 棚卸資産の増加率と売上高成長率。
- 営業キャッシュフローと純利益の乖離。
- 資産計上される支出の急増。
- 一時利益、公正価値評価、税率変化。
- SBCと発行済株式数。

#### 4. 貸借対照表

- net cash / net debt。
- 売上債権・在庫回転日数。
- のれん・無形資産と減損risk。
- リース負債、偶発債務、流動性。

#### 5. valuation

```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-market-cap \
  --price {価格} --shares {発行済株式数} --reported {報告時価総額} --currency {通貨}

python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation \
  --price {価格} --eps {EPS} --bvps {1株当たり純資産}

python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario \
  --price {価格} --eps {EPS} --shares {発行済株式数} \
  --growth {楽観} {基準} {悲観} \
  --pe {楽観PER} {基準PER} {悲観PER}
```

価格、通貨、株式数、基準日、会計期間をそろえる。

**出力**：すべての計算に入力値とツール出力を付け、利益品質を🟢 / 🟡 / 🔴 / ⚪で示す。バフェットの観点から要約するが、架空の直接引用をしない。

---

### Agent 3：競争構造 — マンガーの観点

**中核問い：今回の決算から、競争構造のどの変化を確認できるか。**

#### 1. 決算数値から推定する競争変化

- 売上高成長率と業界成長率。
- 売上総利益率と価格競争。
- marketing・sales費用率と顧客獲得負担。
- 研究開発支出が先行投資か、競合追随か。
- market share、backlog、order、retention。

#### 2. 競合比較

- 同期または近い期間の主要競合を比較する。
- 成長率、margin、capital expenditure、研究開発、KPI。
- 会計期間と通貨をそろえ、比較不能な点を明記する。
- 誰がshareまたはprofit poolを獲得しているか。

#### 3. 経営陣の競争認識

- callで競争をどう説明したか。
- 競合を名指ししたか。事実に基づく説明か。
- 新しい競合、代替技術、platform shiftを認識しているか。

#### 4. 業界signal

- AIその他の技術変化。
- 規制・政策。
- 顧客需要と業界cycle。
- value chainの力関係。

#### 5. 逆向き思考

- この会社を失敗させる経路は何か。
- 今回の決算に、その先行signalはあるか。
- 5年後に転換点と評価される可能性があるか。反証条件は何か。

**出力**：競争構造を強化 / 横ばい / 悪化 / 判定不能で示し、競合比較表と逆向きの結論を付ける。

---

### Agent 4：永久損失risk — 李録の観点

**中核問い：この決算に、永久的な資本損失へつながるsignalまたは説明不足はあるか。**

「経営陣が隠している」と先に決めつけず、開示された事実、説明の不足、確認不能事項を分ける。

#### 1. 語調と説明品質

- 🟢率直：問題と原因を自ら説明する。
- 🟢明確：期限と定量目標を示す。
- 🔴曖昧：検証不能な一般論へ逃げる。
- 🔴論点変更：質問へ直接答えない。
- 🔴外部帰責：内部要因を検討せず、すべて外部へ帰属する。

#### 2. 過去の約束

- 前回の具体的guidanceと今回実績を一つずつ比較する。
- 達成、一部達成、未達、判定不能を示す。
- 未達理由と、説明が事前・事後のどちらかを確認する。

#### 3. 注記

- 関連当事者取引。
- SBC、option、RSUと希薄化。
- 偶発債務、訴訟、保証。
- 会計方針とsegment変更。
- 顧客・supplier集中。
- 税務、規制、資産減損。

#### 4. Q&A

最も重要な3～5問を抽出する。

| 質問 | 回答の要旨 | 回答品質（1～5） | 回避 | 未解決点 |
|---|---|:---:|:---:|---|

#### 5. 永久損失

- 法令・compliance・governance。
- 資本配分による不可逆な損失。
- 過大債務と資金調達。
- 技術・事業modelの構造的崩壊。
- 経営陣の誠実性。

**出力**：経営陣の信頼性を★1～5で評価し、約束の履行状況、risk signal、未確認事項を示す。

## Phase 2：Team Leadが研究底稿を作る

4報告がそろった後、単純に連結せず次を抽出する。

1. **共通点**：複数観点が独立に支持する結論。
2. **矛盾**：事業KPIは改善したが競争費用は上昇する、などの重要なtrade-off。
3. **空白**：4Agentが十分に扱わなかったが、投資仮説に重要な事項。
4. **データ差異**：期間、通貨、会計定義、情報源の違い。

### 研究底稿template

```markdown
# {会社名} {期間} 決算精読

> 調査日：{YYYY-MM-DD}
> データ基準日：{YYYY-MM-DD}
> 資料入手区分：{A/B/C}
> 本資料は調査・学習用であり、投資助言ではない。

## 結論
{50～100字。比較基準、主要変化、投資仮説への影響}

## 最重要の3つの変化
{各100字以内}

## 4観点の評価
| 観点 | 中核問い | 結論 | 評価 | 前回比 |
|---|---|---|---|---|

## 中核データ
| 指標 | 当期 | 比較期間 | 前年同期比 | 会社guidance・市場予想 |
|---|---:|---:|---:|---:|

## 観点別分析
{各3～5点}

## 経営陣の説明と約束
{履行表、Q&A、語調}

## 共通点と矛盾

## 投資仮説への影響
- 強化 / 変化なし / 弱含み / 毀損 / 破綻 / 判定不能
- 根拠：
- 反証条件：

## 次に確認する事項

## 出典・検算記録
```

「4人なら何をするか」という架空の売買指示や直接引用を作らない。必要なら「各枠組みから見た選択肢」として、条件付きで記述する。

## Phase 3：Editor AgentとReader Agent

研究底稿完成後、**同一メッセージで2Agentを並行起動**する。

### Agent 5：Editor Agent

目的は専門性を削らず、公開媒体で読める構成へ変えることである。

#### 1. titleと冒頭

- titleは中核事実を含み、clickbaitにしない。
- 冒頭100字以内で、決算の最重要変化と読む理由を示す。
- 例：`XはAIへ260億元（CNY）を投じた。今回の決算で収益化の兆候は確認できたか。`
- `衝撃`, `暴落確定`など、本文で立証できない表現を避ける。

#### 2. 構成

- inverted pyramidで、重要な3変化を先に置く。
- 表を必要なものへ絞り、長い分析を小見出しと要点へ分ける。
- 約500字ごとに、その節の結論を短く置く。
- 共通点だけでなく、4観点の矛盾を残す。

#### 3. 表現

- 財務用語は正確な日本語で説明する。比喩は補助として使い、会計上の意味を歪めない。
- 例：営業キャッシュフローが純利益より30%少ない場合、「会計上100の利益に対し当期のcash inflowは70」と数値と理由を説明する。
- 段落はmobileで読みやすい長さにする。
- 本人の架空の語録を作らず、4つの枠組みからの要約として表現する。

#### 4. 読者価値

各節について、読者が何を理解し、次に何を確認できるかを明確にする。結論では保有者、未保有者それぞれに、追加調査・確認条件を示すが、断定的な注文指示にはしない。

#### 5. 形式

- WeChat公式アカウントなどmobile媒体で読みやすい見出しと段落。
- 目安1,000～3,000字。ただし重要な留保を削って無理に短縮しない。
- 最終出力は日本語の「だ・である調」とする。

**出力**：完全な公開記事案。

### Agent 6：Reader Agent

想定読者は、価値投資へ関心があり、基礎的な財務知識を持ち、対象企業を保有または監視している一般投資家である。

#### 評価項目

1. **可読性（30%）**
   - 読了時間、飛ばしたくなる段落、理解困難な文、pace。
2. **情報価値（30%）**
   - 理解が深まるか、独自の発見があるか、不要な重複は何か。
3. **信頼性（20%）**
   - 出典、根拠、反対証拠、過度な確信、引用の正確性。
4. **次の確認につながるか（20%）**
   - 保有者・未保有者が、どの指標と事象を次に確認すべきか分かるか。

#### 出力template

```markdown
## Reader Review

### 総合評価：X/10

### 良い点
-

### 必須修正
- 問題：
  修正案：

### 改善候補
-

### 未回答の重要な問い
-

### 一文の評価
```

### Team Leadによる定稿

1. 必須修正を一つずつ処理する。
2. 改善候補は、正確性と可読性に寄与するものを採用する。
3. 未回答の問いは、根拠を確認できる場合だけ追加する。
4. 全文を読み直し、数値、論理、相互参照、出典を確認する。
5. Editor Agentによる簡略化で重要な留保が失われていないか確認する。

## データ監査

最終記事に対して次を行う。

```bash
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report reports/{会社名}/{会社名}-earnings-{期間}.md

python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<入力済みJSON>' \
  --report {報告ファイル名}
```

- **合格**：抽出された数値が許容差内である。
- **差戻し**：不一致を修正し、再確認する。

監査合格だけで正確性が保証されたとは扱わず、未抽出の数値、会社説明、市場予想、定性的結論も確認する。

## 重要原則

- 原文を優先し、要約だけで判断しない。
- 4観点を別々の部門として並べず、互いに検証・反証させる。
- Team Leadは共通点と矛盾を抽出し、単純連結しない。
- 上振れ・一致・下振れは、比較基準と主要項目を明示して結論を出す。
- positiveな発見にも反対証拠を付ける。
- 編集は専門性を低下させず、理解可能性を高める。
- Reader Reviewを形式的な承認にしない。
- 財務計算は`financial_rigor.py`で検算する。
- 事実、会社説明、市場予想、推定、評価を区別する。
- 本資料は調査・学習用であり、投資助言ではない。
