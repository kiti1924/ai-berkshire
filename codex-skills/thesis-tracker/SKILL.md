# 投資仮説トラッカー：購入後の規律を保つ

$ARGUMENTS に対して投資仮説の作成または継続確認を行う。

**対応する入力形式**：

- `会社名`：初回は投資仮説を作成し、既存ファイルがある場合は追跡確認を行う。
- `会社名 仮説作成`：既存の有無にかかわらず投資仮説を再構築する。
- `会社名 四半期確認`：最新決算に基づいて投資仮説を確認する。

> 「購入は始まりにすぎない。保有中の継続的な確認こそが本当の仕事である。」— 李録の投資姿勢を要約した表現
>
> 「事実が変われば、私は考えを変える。」— Keynesに帰される言葉

## 設計思想

多くの投資家は「調査 → 購入 → 祈る」という流れに陥る。購入後の体系的な確認がないと、次の問題が起きる。

- 売却すべきときに「そのうち戻る」と先延ばしする。
- 売却すべきでないときに、株価下落だけを理由に恐慌売りする。
- 当初の購入理由を忘れ、判断基準が後付けになる。

購入前に、投資理由だけでなく「どの事実が変われば売却を検討するか」を書き、決算ごとに仮説の健全性を確認する。

## 実行手順

### 手順1：操作モードを判定する

会社の投資仮説ファイル`reports/{会社名}-thesis.md`が存在するか確認する。

- 存在しない場合：**仮説作成モード**へ進む。
- 存在する場合：**追跡確認モード**へ進む。
- ユーザーが既存ファイルの存在を示したのに見つからない場合：ファイルパスを確認する。

## モードA：投資仮説を作成する

### A0：データを収集する

WebSearchを使い、現在株価、PER、PBR、配当利回りなどの評価指標、最新決算の主要数値を取得する。既存の`/investment-research`または`/investment-team`レポートがあれば、先に読み、利用できる事実と出典を引き継ぐ。

重要な評価データは、次のコマンドで検証する。

```bash
python3 tools/financial_rigor.py verify-valuation ...
```

株価、通貨、単位、基準日、会計期間、株式分割調整の有無を明記する。

### A1：中核となる投資仮説

200字以内を目安に、次の5点を各1文で書く。

```text
私は ___（通貨）で ___社を購入した／購入候補とする。理由は次のとおりである。
1. この事業の本質は___であり、収益が生まれる仕組みを___と理解している。
2. 堀は___であり、現在は拡大／安定／縮小している。
3. 経営陣は___であり、信頼性を示す根拠は___である。
4. 現在価格は推定内在価値の約___%であり、安全余裕は___から生じる。
5. 仮説が一部外れても下方リスクを抑えられる理由は___である。
```

**5文を具体的に書けない場合は、購入判断の根拠が十分に整理されていない。無理に空欄を埋めず、情報不足として扱う。**

### A2：検証可能な前提へ分解する

投資仮説を3～7個の具体的な前提に分解する。少な過ぎれば検討が浅く、多過ぎれば仮説の焦点が定まっていない可能性がある。

| # | 中核前提 | 検証方法 | 確認頻度 | 現在の状態 |
|---|---|---|---|---|
| 1 | 例：売上高成長率が15%以上を維持する | 四半期売上高成長率 | 四半期ごと | 🟢 成立 |
| 2 | 例：売上総利益率が60%以上で安定する | 四半期売上総利益率 | 四半期ごと | 🟢 成立 |
| 3 | 例：経営陣が自社株買いを継続する | 開示資料・キャッシュフロー計算書 | 四半期ごと | 🟢 成立 |
| 4 | 例：競合が同等の能力を獲得していない | 業界データ・競合決算 | 半年ごと | 🟢 成立 |
| 5 | ... | ... | ... | ... |

各前提には、判定に使う指標、閾値、対象期間を可能な限り設定する。

### A3：レッドラインを定義する

レッドラインはいずれかが発生した時点で、投資仮説を直ちに再評価する条件である。行動は自動売買指示ではなく、調査と保有判断を再開するための規律として扱う。

| # | レッドライン | 重大度 | 発生時の対応 |
|---|---|---|---|
| 1 | 例：粉飾決算や不公正な関連当事者取引など、経営陣の誠実性に重大な疑義が生じる | 致命的 | 事実確認後、保有継続の前提を撤回して売却を含め再判断する |
| 2 | 例：中核事業の売上高が2四半期連続で減少する | 重大 | 仮説と企業価値を再評価し、必要なら持分縮小を検討する |
| 3 | 例：競合が同等以上の能力を獲得し、堀が明確に損なわれる | 重大 | 深掘り調査を開始し、撤退を含め判断する |
| 4 | 例：規制変更によって事業モデルの成立条件が根本から変わる | 重大 | シナリオと内在価値を再計算する |
| 5 | 例：経営陣または主要株主が計画外の大規模売却を行う | 警告 | 売却理由、資本配分、ガバナンスを詳しく調べる |

段永平の売却判断は、おおむね「購入判断が誤っていた」「会社の前提が変わった」「より良い機会が見つかった」という3類型で整理できる。

### A4：評価の基準点を記録する

| 指標 | 購入時 | 楽観シナリオ | 基準シナリオ | 悲観シナリオ |
|---|---:|---:|---:|---:|
| 株価（通貨・基準日） | | | | |
| PER（実績・予想の別） | | | | |
| 時価総額 | | | | |
| 推定内在価値 | | | | |
| 安全余裕 | | | | |

推定値には計算式、入力値、通貨、単位、基準日、仮定を付ける。一点の目標値だけでなく範囲を示す。

### A5：仮説ファイルを保存する

`reports/{会社名}-thesis.md`へ次を保存する。

- 作成日とデータ基準日
- 購入価格、通貨、数量またはportfolio比率。未購入なら「購入候補」と記載する
- 5文の中核仮説
- 検証可能な前提一覧
- レッドライン一覧
- 評価の基準点
- 出典
- 空の追跡履歴表

## モードB：投資仮説を追跡確認する

### B1：既存の仮説を読む

`reports/{会社名}-thesis.md`から次を読み込む。

- 中核仮説
- 検証可能な前提一覧
- レッドライン一覧
- 評価の基準点
- 前回確認の記録

### B2：最新情報を収集する

WebSearchと一次資料から次を取得する。

1. 前回確認後に公表された最新決算。
2. 経営陣交代、規制、競争環境、訴訟、資本配分などの重要事象。
3. 現在株価と評価指標。
4. 主要株主・役員の売買記録。
5. 会社側の説明と、外部から確認できる事実の差。

「最新」は調査日現在で確認し、各情報に公表日と対象期間を付ける。

### B3：中核前提を一つずつ検証する

| # | 中核前提 | 前回の状態 | 最新の証拠 | 現在の状態 | 変化 |
|---|---|---|---|---|---|
| 1 | 売上高成長率15%以上 | 🟢 成立 | Q4売上高成長率12% | 🟡 弱含み | ⚠️ |
| 2 | 売上総利益率60%以上 | 🟢 成立 | 売上総利益率61.2% | 🟢 成立 | — |
| 3 | ... | ... | ... | ... | ... |

状態の定義：

- 🟢 **成立**：最新の証拠が前提を支持する。
- 🟡 **弱含み**：許容範囲内だが、傾向が不利になっている。
- 🔴 **毀損**：最新の証拠が前提を明確に支持しない。
- ⚫ **破綻**：前提が覆された。
- ⚪ **判定不能**：必要なデータがない、または証拠が矛盾している。

### B4：レッドラインを確認する

| # | レッドライン | 発生 | 証拠 |
|---|---|:---:|---|
| 1 | 経営陣の誠実性に重大な疑義 | ❌ 未発生 | — |
| 2 | 中核事業が2四半期連続で減収 | ❌ 未発生 | — |

レッドラインが発生した場合は、報告冒頭に明示し、確認済み事実、未確認事項、推奨する再調査または判断手順を示す。

### B5：評価を更新する

| 指標 | 購入時 | 前回確認 | 現在 | 変化 |
|---|---:|---:|---:|---:|
| 株価 | | | | |
| PER（TTM） | | | | |
| 推定内在価値 | | | | |
| 安全余裕 | | | | |

異なる通貨、株式分割前後、ADRと原株、実績PERと予想PERを混同しない。

### B6：追跡報告を出力する

#### 報告構成

```text
1. 結論とレッドライン
2. 投資仮説の健全性スコア
3. 中核前提の確認結果
4. レッドラインの確認結果
5. 今期の重要な変化（500字以内）
6. 評価の更新
7. 判断上の選択肢と根拠
8. 次回確認事項と時期
9. 出典
```

#### 健全性スコア

```text
健全性 = 10
         -（⚫ 破綻した前提数 × 3）
         -（🔴 毀損した前提数 × 2）
         -（🟡 弱含みの前提数 × 1）
         -（発生したレッドライン数 × 5）
```

最低1点、最高10点とする。⚪判定不能は機械的に減点せず、情報不足として別に警告する。レッドラインの重大度によっては、点数にかかわらず仮説を破綻扱いとする。

| スコア | 意味 | 判断上の選択肢 |
|:---:|---|---|
| 9～10 | すべての主要前提が成立し、購入時より強まっている | 評価と集中リスクを確認したうえで追加投資を検討できる |
| 7～8 | 中核前提は成立しているが、一部が弱含み | 継続保有を基本に、弱含みの指標を重点確認する |
| 5～6 | 1～2個の前提が毀損したが、中核論理は残る | 保有比率と下方リスクを再評価する |
| 3～4 | 複数の前提が毀損し、仮説の基礎が揺らぐ | 持分縮小または撤退を含め再判断する |
| 1～2 | レッドライン発生または中核前提の破綻 | 投資仮説を撤回し、売却を含む対応を優先的に検討する |

#### 結論で必ず答えること

1. **投資仮説は維持できるか**：維持 / 弱含み / 毀損 / 破綻 / 判定不能。
2. **次に何をするか**：追加調査、継続保有、持分縮小、撤退検討など、選択肢と根拠を示す。個別事情を無視した断定的な売買指示にはしない。
3. **次回確認時期**：次の決算公表後、特定事象の発生後、または具体的な日付。

### B7：仮説ファイルを更新する

今回の確認結果を`reports/{会社名}-thesis.md`の追跡履歴へ追記する。

| 確認日 | 健全性 | 主な変化 | 判断上の選択肢 |
|---|:---:|---|---|
| 2026-04-09 | 7/10 | 売上高成長率は12%へ鈍化したが、利益率は改善 | 継続保有、次四半期を重点確認 |

既存記録を上書きせず、今回追加した事実と評価を区別する。

## 重要原則

- **購入前に売却条件を書く**：冷静な時点で決めた基準を、恐怖や高揚の中で後付けしない。
- **検証可能な仮説にする**：「良い会社」ではなく、「ROEが25%を超え、悪化傾向にない」のように指標と期間を示す。
- **レッドラインを先延ばししない**：発生時は事実を確認し、直ちに再評価する。
- **仮説の破綻と株価下落を分ける**：株価が30%下落しても仮説が維持される場合があり、株価が上昇していても仮説が破綻する場合がある。
- **誤りを認める**：当初仮説が誤っていた場合は、面子のために維持しない。
- **事実と評価を分ける**：会社発表、市場予想、計算上の推定、分析者の判断を明示的に区別する。
- **本資料は調査・学習用であり、投資助言ではない。**
