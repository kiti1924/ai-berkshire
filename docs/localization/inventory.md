## 2. 集計方法

追跡対象は`git ls-files -z`でNUL区切り取得した。パス名に中国語が含まれていてもGitの引用表示に依存しない方法である。

本文の言語候補は、次の2方法で照合した。

1. Unicode CJK統合漢字（U+3400–U+4DBF、U+4E00–U+9FFF）を1文字以上含むUTF-8テキストを検出した。
2. 簡体字で使用される異体字集合と、中国語文で頻出する語句を用いて簡体字シグナルを検出した。

両方法とも2,190ファイルとなった。これは本基準SHAでは中国語本文が支配的であり、日本語本文がまだ導入されていないためである。ただし、漢字だけでは日本語と中国語を完全には区別できない。第2方法もヒューリスティックであり、短い固有名詞だけのファイルでは見落とし得る。画像12ファイルはOCRしていない。

## 6. 後方互換性のため保持すべき中国語トークン

日本語出力へ移行しても、次は機械的に全置換しない。

1. **情報検索語と情報源名**
   - 雪球、东方财富、同花顺、巨潮资讯、港交所公告等
   - 中国語資料を検索するための会社名、製品名、役職名、規制用語
2. **市場・証券・通貨・単位**
   - A股、港股、沪深300、人民币、港元、亿元、万亿等
   - 特に`亿元`は「億円」へ変換せず、通貨を伴う単位として解釈する必要がある。
3. **パーサー・監査用トークン**
   - 中国語の句読点、桁区切り、財務見出し、会計用語
   - `tools/report_audit.py`等が既存中国語レポートを解析するために必要な文字列
4. **既存ファイル名とリンク先**
   - `reports/`以下の会社名、テーマ名、Markdownリンク
   - 参照切れを避けるため、別Phaseで移行方針を決めるまで維持する。
5. **引用・原資料の表記**
   - 経営者発言、法定開示の原題、媒体名、一次資料内の表現
   - 日本語説明を付けても原文確認可能性を残す。
6. **外部仕様値**
   - APIレスポンス、JSONキー、コマンド、ticker、取引所コード、URL、既存fixtureの期待値
