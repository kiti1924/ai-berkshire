# 業界funnel screening：全市場から3社へ絞る価値投資process

$ARGUMENTS の業界または投資themeについて、全市場から段階的に候補を絞り、最終的に最大3社を深掘り対象として選ぶ。

## 対象となる依頼

`AI計算資源`、`新薬`、`robotics`などの業界・方向性について、次を行う。

1. 中国本土A株、香港株、米国株、国際市場、重要な非上場候補を広く確認する。
2. 共通基準でstoryだけの企業や品質不足の企業を除外する。
3. 深い調査に値する最大3社へ集中する。
4. 各段階の採用・除外理由を残し、後から再現できるようにする。

`industry-research`はvalue chain構造と全体像を重視し、本Skillは企業screeningを重視する。先に`industry-research`で構造を把握し、本Skillで候補を絞る方法を推奨する。

---

## funnel全体

```text
第1層：全市場scan       30～60社
      （売買活発度、値動き、時価総額上位の和集合）
        ↓ 価値投資の5基準
第2層：初期screening      10社以下
      （5基準を概ね満たし、堀が★★★以上）
        ↓ structured analysis
第3層：詳細比較            10社以下
      （1社300～500字）
        ↓ 最終選定
第4層：4観点の深掘り        最大3社
      （1社800～1,200字）
```

除外した企業も名称と理由を記録し、black boxにしない。

---

## 手順1：全市場scan

### 1.1 活発な企業の定義

次の3集合の和を作る。

**A・売買活発度**

- 直近30日の日次平均売買代金が業界上位。中国本土、香港、米国など市場ごとに上位30社を参考とする。

**B・価格変動**

- 直近30日の上昇率上位20社。
- 直近90日の上昇率上位20社。
- 両者の和集合。

**C・時価総額**

- 業界内の時価総額上位30社。直近の上昇・下落にかかわらず含める。

最終poolはA ∪ B ∪ Cとし、30～60社を目安とする。上昇率だけで魅力度を判断せず、survivorship biasとmomentum biasを記載する。

### 1.2 必ず確認する市場

| 市場 | 主な探索先 |
|---|---|
| 中国本土A株 | Eastmoney（东方财富）、同花顺、取引所、企業開示 |
| 香港株 | HKEX、aastocks、企業開示 |
| 米国株 | NYSE、NASDAQ、業界ETF構成、SEC提出資料 |
| 国際市場 | 日本、韓国、台湾、欧州、豪州など。特にsemiconductor、electronicsでは省略しない |
| 非上場 | 重要な競合・supplierを「将来IPO候補」または「直接投資不可」として別表にする |

情報源のavailabilityに偏らず、英語、中国語、日本語その他の正式名称で検索する。

### 1.3 scan表

| 会社 | ticker | 市場 | 時価総額 | 事業を一文で | 対象業界売上比率 | 採用経路（A/B/C） | 情報量 |
|---|---|---|---:|---|---:|---|---|

時価総額には通貨と基準日時を付ける。

**自己点検**：

- 対象業界売上比率が30%未満なら「pure playではない」と明記する。
- 英語資料が少ないことを理由に中国・アジア企業を落とさない。
- 小型企業を検索結果の少なさだけで落とさない。
- 非上場企業のvaluationやIPO時期は、確認済み公表と市場推測を分ける。

## 手順2：価値投資の5基準で初期screeningする

30～60社へ共通基準を適用し、原則として10社以下へ絞る。

### 2.1 5基準

| # | 指標 | 基本基準 | 例外・調整 | 主な情報源 |
|---|---|---|---|---|
| 1 | PER・valuation | 自社historical rangeと同業に対して妥当 | 高成長ではPEG 1.5未満を参考にするが、成長の持続性を別途確認 | 法定開示、価格data、信頼できる金融data |
| 2 | ROE | 15%超、または直近3年で改善 | asset-heavy industryはROICやcycle平均も使う | 財務諸表 |
| 3 | 営業キャッシュフロー | プラスかつ純利益の70%超 | 一時的な運転資本変動を説明する | cash flow statement |
| 4 | 負債比率 | 60%未満 | utility・電力は70%を参考に調整する | balance sheet |
| 5 | 経済的な堀 | ★★★以上 | 例外なし。証拠不足は不明とする | 定量・定性の両方 |

堀の類型は、ブランド・価格決定力、切替コスト、ネットワーク効果、規模の経済、技術・license・resource・certificationである。

銀行、保険、REITなどでは一般企業と同じ負債・cash flow基準を機械的に使わず、業種固有の指標へ置き換え、変更理由を記載する。

### 2.2 screening表

| 会社 | PER | ROE | OCF / 純利益 | 負債比率 | 堀 | 総合 | 採用・除外 | 理由 |
|---|---:|---:|---:|---:|---|---|---|---|

**採用規則**：

- 5項目すべて満たす：採用。
- 4項目を満たし、残る1項目が閾値付近または業種上合理的：yellowとして採用。
- 4項目未満：除外し、具体的な指標を示す。
- データ不足：自動的に採用も除外もせず、「判定保留」とする。最終3社には入れない。

12社を超える場合は、堀を★★★★以上へ引き上げるだけでなく、情報品質、cash conversion、valuationを再比較する。

## 手順3：詳細比較

初期screeningを通過した各社を300～500字で整理する。

```markdown

**事業モデル**：
{何を、誰に、どのように販売し、どう課金するか}

**財務品質**：
- 売上高成長率 / 利益成長率 / 売上総利益率 / ROE / cash flow
- 直近1～2年の重要な転換点

**経済的な堀**：
- 主な堀と具体的証拠
- 5年後の持続性と反証条件

**主要risk**：
1.
2.
3.

**valuation**：
- PER / PSR / EV/EBITDA / FCF Yieldとhistorical range
- 同業比較
- 結論：割高 / おおむね妥当 / 割安 / 判定不能

**最終候補**：採用 / 非採用 / watchlist
**理由**：
```

### 最大3社の選び方

単純な点数上位ではなく、riskとreturn sourceの違いを考える。

- 少なくとも1社：予測可能性が比較的高く、upsideは穏やかなcore候補。
- 少なくとも1社：中程度の予測可能性と成長余地を持つcandidate。
- 必要な場合のみ1社：高いoptionalityと高い損失riskを持つcandidate。

十分な企業が3社なければ、`2社 + watchlist 1社`または`1社のみ`とする。数を合わせるために品質基準を下げない。

## 手順4：最大3社を4観点で深掘りする

各社800～1,200字を目安とする。

### 4.1 段永平 — 事業の本質

- 一文で事業を説明する。
- 優れた事業か。理由と反対証拠は何か。
- 企業の「本分」と顧客価値は何か。経営陣は逸脱していないか。
- 収益modelの持続性はどこにあるか。

### 4.2 バフェット — 経済的な堀と安全余裕

| 堀 | 強さ（★1～5） | 証拠 | 方向 |
|---|---|---|---|
| ブランド・価格決定力 | | | |
| 切替コスト | | | |
| ネットワーク効果 | | | |
| 規模の経済 | | | |
| 技術・license障壁 | | | |

- 10年後も堀は残るか。
- 現在価格の安全余裕はどこから生じるか。
- 重要なvaluation仮定を示す。

### 4.3 マンガー — 失敗経路

- 最も現実的な失敗経路を3件。
- 悲観scenarioの価値範囲。
- 合理的な投資家が買わない理由。
- ethics、compliance、経営陣risk。

### 4.4 李録 — 長期的変化

- 長期的なparadigm shiftか、一時的な投資cycleか。
- 最も近い歴史的類似例と相違点。
- 10～20年後のindustry structureと対象企業の位置。
- winner-takes-mostか、競争が分散するか。

### 4.5 統合評価

```text
調査優先度：★★★★☆
役割：core / satellite / optionality / watchlist
価格条件：現在 / {条件を満たす調整後} / より良い価格を待つ
テーマ内の参考比率：{範囲。ユーザー固有条件が不明なら参考値と明記}
監視指標：{仮説を強化・毀損するsignal}
```

具体的な注文数量や断定的な売買指示にせず、scenarioと条件を示す。

## 手順5：統合出力

### 5.1 最大3社の比較

| 会社 | 役割 | 優先度 | テーマ内参考比率 | 中核logic | 最大risk |
|---|---|---|---:|---|---|
| A | core | ★★★★★ | 50～60% | | |
| B | satellite | ★★★★☆ | 25～35% | | |
| C | optionality | ★★★☆☆ | 5～15% | | |

比率は例示であり、総portfolioに占めるtheme上限、他資産との相関、通貨、流動性を別途確認する。

### 5.2 ETF代替

個別企業を選ばない選択肢として、中国本土、香港、米国などから関連ETFを1～3件示す。基準日時点の構成銘柄、expense ratio、流動性、通貨、対象indexを確認する。

### 5.3 industryの位置

- PER、PBRその他適切なvaluationのhistorical percentile。
- fund flow。Stock Connect、ETF creation/redemption、sell-side coverageなどの時間差を明記する。
- early / expansion / mature / declineのどこか。根拠と反証条件を示す。

### 5.4 情報充足度

| 観点 | A/B/C | 説明 |
|---|---|---|
| 財務data | | |
| valuationの鮮度 | | |
| industry structure | | |
| 経営陣 | | |

- A：一次資料と複数情報源が十分である。
- B：一部不足するが、主要結論には大きく影響しない。
- C：不足が大きく、結論の確度を下げる。

### 5.5 更新待ち

推定値、未確認値、次回決算で確認する指標、今後の公表予定を明示する。

### 5.6 情報源

法定開示、企業資料、公的統計、調査資料、報道に分け、重要数値と対応づける。

## AI調査のbias

| bias | 起こり方 | 対応 |
|---|---|---|
| 大型株bias | 資料量と分析文量が多く、良く見える | 同じ指標と堀の証拠で評価し、文量でrankingしない |
| 英語bias | 米国企業を過大評価し、中国本土・香港企業を見落とす | 英語・中国語・日本語で検索し、A/H企業を省略しない |
| story bias | 値上がりとmedia露出を実収益と混同する | theme関連売上比率、顧客、cash flowを確認する |
| 現在値bias | 現在の財務が良い企業だけを残す | 改善trendを例外条件として扱うが、turnaroundの証拠を要求する |
| 上場bias | 非上場leaderを無視する | 直接投資可能性と分けて重要な非上場企業を記録する |

## 出力要件

1. 保存先：`reports/{業界名}-funnel-{YYYYMMDD}.md`
2. 最終出力は自然な日本語の「だ・である調」とする。
3. 結論を先に示し、不要な修辞を避ける。
4. 数値へ情報源、通貨、単位、基準日を付け、推定値は「推定」とする。
5. データ、logic、結論の順で示す。
6. 中核判断には反対証拠を付ける。
7. 各層の除外企業と理由を残す。
8. 本資料は調査・学習用であり、投資助言ではないと明記する。

## データ監査

```bash
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report <報告ファイルパス>

# skills/financial-data.mdに従い、抽出項目を再取得する

python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<入力済みJSON>' \
  --report <報告ファイル名>
```

- **合格**：抽出数値が許容差内である。
- **差戻し**：不一致を修正し、再監査する。

監査合格だけで企業poolの網羅性、通貨、会計期間、定性的評価の正確性が保証されたとは扱わない。
