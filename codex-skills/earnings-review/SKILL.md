# 決算精読：一次資料を深く読む

$ARGUMENTS に対して決算資料の精読を行う。

**対応する入力形式**：`会社名 期間`。例：`Tencent 2025Q4`、`PDD 2025年次報告書`、`Meituan 最新`。期間指定がなければ、調査日時点で公表済みの最新決算を対象とする。

> 「売り手側の要約ではなく、原資料を読む」という李録（リー・ルー）の調査姿勢を重視する。
>
> 「毎日大量に読み、知識を複利のように積み上げる」というバフェットの姿勢を参考にする。

## 設計思想

多くのAI投資調査は、報道、証券会社レポートの要約、データサイトなど二次情報へ依存する。しかし、投資判断の基礎は、年次報告書、四半期報告書、決算説明資料、earnings callのtranscriptなどの**一次資料**である。

二次情報には次の問題がある。

- 選択されている：作成者の見解に都合の良い数値だけが強調される場合がある。
- 時間差がある：市場参加者が解釈した後の情報になりやすい。
- 文脈が失われる：「売上高15%増」だけでは、成長の質や経営陣の説明が分からない。
- 会計上の定義が省略される：GAAP、IFRS、Non-GAAP、継続事業、為替影響などが混在し得る。

本Skillは一次資料を直接読み、数字だけでなく、経営陣の説明、過去の約束、注記、会計上の変化を確認する。

## 実行手順

### 事前工程：資料の入手状況を評価する

| 区分 | 入手状況 | 分析への影響 |
|---|---|---|
| A | 10-K、10-Q、年次報告書、決算説明資料、transcriptなど原文を十分に入手した | 全工程を実行する |
| B | 原文の一部だけを入手し、残りは信頼できる二次情報で補う | 「一部は二次情報」と明記し、注記分析の確度を下げる |
| C | 報道とデータサイトの要約しかない | 中核財務データの変化へ絞り、注記を推測しない。「一次資料不足」と明記する |

対象資料のタイトル、公開日、対象期間、URLまたは取得元を一覧にする。

### 手順1：一次資料を集める

Taskツールで複数のバックグラウンドAgentを**並行起動**し、次を取得する。

1. **法定開示・決算原文**：企業IR、SEC EDGAR、HKEXnews（披露易）、CNINFO（巨潮资讯）など。
2. **earnings callのtranscriptまたは録音**：企業IRを優先し、必要に応じてSeeking Alpha、雪球（Xueqiu）などを補助的に使う。
3. **株主向け書簡**：年次報告書に含まれる場合は全文を読む。
4. **Investor Day / Analyst Day資料**：対象期間に関連する公表資料があれば読む。
5. **前回決算資料**：経営陣の過去のguidanceと約束を確認するために取得する。

完全な原文を入手できない場合は、`skills/financial-data.md`に従い、複数の情報源で補完する。中国企業を調べる際は、正式名称や中国語検索語を検索に使ってよい。ただし、最終出力は共通locale設定に従う。

### 手順2：中核財務データを抽出し、検証する

#### 2.1 売上高と損益計算書

| 指標 | 当期 | 比較期間 | 前年同期比 | 会社guidance・市場予想 | 判定 |
|---|---:|---:|---:|---:|---|

必ず次を確認する。

- 売上高とsegment別・地域別の内訳。
- 売上総利益と売上総利益率。
- 営業利益と営業利益率。GAAP / IFRSとNon-GAAPを区別する。
- 親会社株主に帰属する当期純利益。非継続事業、評価益、減損など一時項目を分ける。
- 基本EPSと希薄化後EPS。
- 為替一定ベースを会社が示す場合は、実際の報告値と分ける。

「市場予想を上回った」という表現には、予想の情報源、集計時点、比較対象を付ける。

#### 2.2 キャッシュフロー計算書

| 指標 | 当期 | 比較期間 | 変化 | 確認事項 |
|---|---:|---:|---:|---|

必ず次を確認する。

- 営業キャッシュフロー / 純利益。100%超を良好、80%未満を警戒の参考とするが、運転資本や業種差を確認する。
- 設備投資と、その維持投資・成長投資の内訳。開示がなければ推定と明記する。
- FCF = 営業キャッシュフロー - 設備投資。会社独自のFCF定義があれば併記する。
- 自社株買い、配当、債務返済、買収支出。
- 期末の現金及び現金同等物、短期投資。

#### 2.3 貸借対照表

必ず次を確認する。

- 現金・短期投資と有利子負債。
- net cashまたはnet debtの推移。
- 売上債権回転日数。売上計上を支えるため信用条件を緩めていないか。
- 棚卸資産回転日数。需要鈍化や陳腐化の兆候がないか。
- のれんと無形資産。減損リスク、買収依存を確認する。
- 契約負債、前受収益、繰延収益など、事業モデル上重要な項目。
- リース負債、保証、off-balance-sheetの義務。

#### 2.4 重要数値を検算する

`tools/financial_rigor.py`を使う。以下は例であり、実際のサブコマンド仕様へ合わせる。

```bash
# 売上高、純利益などを2情報源以上で比較する
python3 tools/financial_rigor.py cross-validate \
  --field "revenue" \
  --values '{"企業決算": 108.3e9, "独立情報源": 107.9e9}' \
  --unit USD

# 時価総額を検算する
python3 tools/financial_rigor.py verify-market-cap \
  --price 101 --shares 1.488e9 --reported 1.44e11 --currency USD

# 評価指標を検算する
python3 tools/financial_rigor.py verify-valuation \
  --price 101 --eps 9.6 --bvps 26.5 --fcf-per-share 10.2
```

実行前に`--help`または実装を確認し、存在しない引数を作らない。各数値の通貨、単位、対象期間、会計基準をそろえる。

### 手順3：経営陣による説明を精読する

MD&Aとearnings callを段落単位で読み、**何を説明したか**と**どのように説明したか**を分ける。

#### 3.1 語調と説明品質

| signal | 具体的な特徴 | 記録例 |
|---|---|---|
| 🟢 **率直** | 問題を自ら認め、具体的な原因と対応を示す | 「当四半期のmargin低下はXへの投資が想定を上回ったためである」 |
| 🟢 **明確** | 戦略、期限、指標を具体的に示す | 「今後12か月でX事業のshareを15%から20%へ引き上げる」 |
| 🔴 **曖昧** | 「長期的には」「自信がある」など、検証困難な表現だけが続く | 「将来に強い自信を持っている」 |
| 🔴 **論点変更** | 直接質問へ答えず、別の良い指標へ話を移す | marginの質問に売上高成長だけで答える |
| 🔴 **外部要因への過度な帰責** | 問題をすべてマクロ、規制、競合へ帰属させる | 「マクロ環境の影響による」とだけ説明する |

語調だけで不誠実と断定せず、複数期間の発言と実績を確認する。

#### 3.2 過去の約束を追跡する

前回までの決算資料とcallから、具体的なguidanceや約束を抽出し、実績と比較する。

| 過去の説明・約束 | 今回の実績 | 判定 | 差異の説明 |
|---|---:|---|---|
| 「下期にmarginをX%へ回復させる」 | 実績Y% | ✅達成 / ❌未達 / ⚠️一部達成 | |

段永平の経営者評価を参考に、言葉より、過去に述べたことを実行したかを見る。

#### 3.3 Q&Aの重要論点

analystが投げた厳しい質問と、経営陣の回答を抽出する。

| 質問 | 回答の要旨 | 回答品質（1～5） | 回避の有無 | 未解決点 |
|---|---|:---:|:---:|---|

### 手順4：注記と見落とされやすい情報を調べる

#### 4.1 必須確認項目

- [ ] **関連当事者取引**：主要株主、経営陣、関連会社との条件は公正か。
- [ ] **株式報酬**：option、RSUなどの費用と希薄化。行使価格、vesting条件、株式数を確認する。
- [ ] **偶発債務**：訴訟、保証、購入契約、規制案件。
- [ ] **会計方針の変更**：収益認識、耐用年数、資産計上、segment定義など。
- [ ] **segment情報**：事業別の収益性と、収益性の高い事業が低い事業を補助していないか。
- [ ] **顧客・supplier集中**：上位顧客・supplierの比率と変化。
- [ ] **税務・為替**：実効税率、繰延税金資産、為替換算の影響。

#### 4.2 異常signal

- [ ] 売上債権の伸びが売上高を持続的に上回る。
- [ ] 棚卸資産の伸びが売上高を持続的に上回る。
- [ ] 営業キャッシュフローが純利益を下回り、その差が拡大する。
- [ ] 資産計上される支出が急増し、費用認識が先送りされる可能性がある。
- [ ] 一時利益や公正価値評価益への依存が高まる。
- [ ] 調整後利益と法定利益の差が拡大し続ける。
- [ ] 発行済株式数が、自社株買い実施にもかかわらず増えている。

signalは調査開始点であり、不正の断定ではない。注記と経営陣の説明で原因を確認する。

### 手順5：過去と比較する

#### 5.1 時系列

少なくとも直近4四半期、年次報告書なら3年の推移を示す。

| 指標 | Q-4 | Q-3 | Q-2 | Q-1 | 当期 | 傾向 |
|---|---:|---:|---:|---:|---:|---|

重点確認：

- 利益率は改善か悪化か。
- 売上高成長は加速か減速か。
- キャッシュ創出力は改善か悪化か。
- 設備投資負担は上昇か低下か。
- 株式数と1株当たり指標はどう変化したか。

#### 5.2 guidanceとの比較

| 指標 | 過去の会社guidance | 実績 | 差異 | 解釈 |
|---|---:|---:|---:|---|

会社guidance、市場予想、独自予想を混同しない。

### 手順6：精読報告を作る

#### 報告構成

```text
1. 結論
2. 対象資料と資料入手区分
3. 中核データ一覧
4. 当期で最も重要な3つの変化（500字以内）
5. 損益、cash flow、貸借対照表の分析
6. 経営陣の語調と約束の履行
7. 注記で確認した重要事項
8. earnings callの主要Q&A
9. 投資仮説への影響
10. 次回確認事項
11. 出典とデータ検証記録
```

#### 結論で必ず答えること

1. **今回の結果は、会社guidanceまたは明示した市場予想に対して上振れ、概ね一致、下振れのどれか。** 比較基準と重要項目を示す。
2. **投資仮説への影響**：強化 / 変化なし / 弱含み / 毀損 / 破綻 / 判定不能。
3. **次の重要な確認事項**：決算、製品、規制、marginなど、具体的なtriggerを示す。
4. **保有者が再検討すべき選択肢**：継続保有、追加調査、比率見直し、撤退検討などを、根拠と条件付きで示す。個別事情を知らず、断定的な注文指示にしない。

### 手順7：報告を保存する

`reports/{会社名}/{会社名}-earnings-{期間}.md`へ保存する。例：`reports/Tencent/Tencent-earnings-2025Q4.md`

### 手順8：データ監査を行う

```bash
# 手順1：抽出候補を作る
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report reports/{会社名}/{会社名}-earnings-{期間}.md

# 手順2：skills/financial-data.mdに従い、抽出項目を再取得する

# 手順3：判定する
python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<入力済みJSON>' \
  --report {報告ファイル名}
```

- **合格**：抽出された数値が許容差内である。
- **差戻し**：不一致がある。通貨、単位、期間、会計定義、情報源を修正し、再監査する。

監査結果だけで正確性を保証されたと判断せず、未抽出の数値、注記、定性的結論も確認する。

## 重要原則

- **要約より原文を優先する**：可能な限り一次資料へ戻る。
- **絶対値だけでなく変化を見る**：時系列と事前guidanceを比較する。
- **内容と語り方の両方を見る**：ただし語調だけで断罪しない。
- **本文だけでなく注記を読む**：会計方針、希薄化、偶発債務、segmentを確認する。
- **結論を先に示す**：決算の再述ではなく、投資仮説がどう変わったかを示す。
- **事実、会社説明、市場予想、推定、評価を分ける。**
- **最終出力は自然な日本語の「だ・である調」とする。**
- **本資料は調査・学習用であり、投資助言ではない。**
